# Stock before/after stored as -1.0 sentinel in ReminderEvent

`ReminderEventEntity` stores `stockBefore` and `stockAfter` as `Double` columns defaulting to `-1.0` rather than nullable columns. `-1.0` means "stock not tracked for this event." Display logic shows the stock change hint only when `stockBefore != stockAfter` — this single condition handles all cases: old migrated rows (`-1.0 == -1.0`), medicines without stock management (never written, stay equal), events where the dose was taken then un-taken (stock restored → `stockAfter` is written back to equal `stockBefore`). Nullable columns were considered but rejected because Room's AutoMigration handles non-null columns with defaults more predictably than nullable doubles, and the equality check is simpler than a null guard at the call site.
